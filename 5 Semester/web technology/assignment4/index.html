Overview

We think the website in general is easily navigable as it is simple and not very rich in diverse content. While the sections are in an optimal order, changing their order will not render them incomprehensible as they are all mostly independent of each other. We have also used some colors in certain areas to make content even more easily viewable. Some people with disabilities may also able to easily navigate our website via keyboard, eye-tracking or speech navigation without encountering any issues.

Accessability highlights

We think some of the most relevant guidelines are 1.4.1 because we made sure to use color to make content more distinguishable and 2.4.3 because no matter what order the webpage sections are viewed in, their meaning does not change.

Improvement plan

  • in addition to the form submission error text, add a sound for extra feedback

  • fix the http post request to the server so, that data can be send. Possible solutions:
    • check if the send data is in JSON format (set request type to "application/json")
    • check if the server recieved the request
    • check what type of request the server is recieving (also check network activity)
  • add code to prevent the default event handling (preventDefault)
  • adjust file names to the naming convention

Nice to have changes:

  • make assignment 2 work with the created web server from assignment 3
  • for dynamically filling the table, use a different way to execute the js code and do not use the "script" tag
  • delete all "br" tags
  • instead of using IDs for the navigation links, use class
  • correcly adjust the form tags (change from type "img" to "url")
